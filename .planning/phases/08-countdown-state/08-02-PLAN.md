<objective>
Render the client-side cinematic for the server-driven countdown: lock movement AND spell input, snap the camera out to 0.6x zoom and animate it back to the play zoom over 3 s, show the `3 → 2 → 1 → FIGHT!` overlay text driven by inbound `match:countdown-tick` events, and release both locks on `match:state-changed: ACTIVE` so combat unlocks simultaneously on every client.

Purpose: Phase 8 of v1.2 — Plan 08-02 covers LFC-06 (input lock — including the FireBreath/EarthWall gap that `controls.isMovementLocked` alone doesn't close), LFC-07 (camera zoom animation), LFC-08 (overlay rendering — driven by server tick broadcasts, not a client clock), and the client half of LFC-09 (simultaneous unlock). Plan 08-01 (server) is the prerequisite: the server emits the new `match:countdown-tick` events and the genuine `match:state-changed: ACTIVE` broadcast at t+3500 ms.

Output: A new `MatchCountdownTickPayload` type mirrored on the client. A new `NETWORK_MATCH_COUNTDOWN_TICK` event on `CUSTOM_EVENTS`. `NetworkManager` re-emits the inbound socket event onto the bus. `GameScene` gains a `#combatLocked` field, an `#enterCountdownMode` / `#exitCountdownMode` pair driven by `NETWORK_MATCH_STATE_CHANGED`, an `#onCountdownTick` handler driven by `NETWORK_MATCH_COUNTDOWN_TICK`, top-of-function guards on `#updateFireBreathChanneling` and `#updateEarthWallSpell`, and a single in-scene `setScrollFactor(0)` text overlay. The final task is a checkpoint:human-verify two-window UAT that exercises LFC-06..09 visually because the rendering side is not unit-testable in this repo.
</objective>
