# Plan 02-01: Type System + Server Team Protocol

**Goal:** Add `team` field to `PlayerInfo` everywhere and wire a server-authoritative `lobby:assign-team` socket event so team assignments propagate to all clients before match start.
**Estimate:** S
**Depends on:** none
**Delivers:** PLR-04 (server foundation â€” type system + socket protocol)

## Context

`PlayerInfo` in both `src/networking/types.ts` and `game-server/src/types.ts` has no `team` field. There is no mechanism for the lobby host to assign players to teams. Adding `team?: number` (optional to avoid breaking existing code paths) plus a server-authoritative socket event `lobby:assign-team` unblocks the lobby UX in Plan 02-02. Because team assignments are stored on `PlayerInfo` objects inside the live lobby, they flow automatically into `matchConfig.players[i].team` when `lobby:start` is processed â€” no changes to the start flow are needed.

### Task 2: Add `setPlayerTeam()` to `LobbyManager`

**File:** `game-server/src/lobby-manager.ts`
**Action:** Edit

Add a new public method `setPlayerTeam` after the existing `startLobby` method. Follow the same host-guard pattern used by `setMode()`:

```typescript
setPlayerTeam(requesterSocketId: string, targetPlayerId: string, team: number): Lobby | null {
  const lobby = this.getLobbyBySocketId(requesterSocketId);
  if (!lobby) return null;
  const requester = lobby.players.find(p => p.socketId === requesterSocketId);
  if (!requester || requester.id !== lobby.hostPlayerId) return null;  // host-only
  const target = lobby.players.find(p => p.id === targetPlayerId);
  if (!target) return null;
  target.team = team;
  return lobby;
}
```

The guard `requester.id !== lobby.hostPlayerId` mirrors the guard in `setMode()` and prevents non-hosts from assigning teams even via devtools.

### Task 3: Add `lobby:assign-team` handler in `server.ts`

**File:** `game-server/src/server.ts`
**Action:** Edit

Add a new socket event handler immediately after the `socket.on('lobby:set-mode', ...)` block (around line 52):

```typescript
socket.on('lobby:assign-team', ({ targetPlayerId, team }: { targetPlayerId: string; team: number }) => {
  const lobby = lobbyManager.setPlayerTeam(socket.id, targetPlayerId, team);
  if (lobby) io.to(`lobby:${lobby.id}`).emit('lobby:updated', { lobby });
});
```

The `lobby:updated` broadcast reuses the existing client-side `#onWaitingRoomUpdate` â†’ `#renderPlayerList` path â€” no new client event handler required.
