# Plan 02-02: LobbyScene Host Fix + Team Assignment UI

**Goal:** Fix the START button being shown to all clients (host detection bug), and add team assignment UI to the waiting room — host gets toggle buttons, non-hosts get read-only badges.
**Estimate:** M
**Depends on:** 02-01 (requires `sendLobbyAssignTeam()` on `NetworkManager` and `team` on `PlayerInfo`)
**Delivers:** PLR-01 (no hardcoded role assumptions in client), PLR-04 (N-player teams configurable before match start)

---

## Context

Two bugs exist in `src/scenes/lobby-scene.ts`:

**Bug 1 — START button shown to all clients** (`#showWaitingRoomView`, line ~202):
```typescript
if (lobby.hostPlayerId === nm.localPlayerId || lobby.players[0]?.socketId) {
```
- `nm.localPlayerId` is an empty string during the waiting room — it is only set inside the `lobby:started` handler, which fires at match start, not at lobby join time.
- `lobby.players[0]?.socketId` is a non-empty string for any occupied lobby, so the `||` short-circuits and the condition is always `true`. Every client sees the START button.

**Bug 2 — No team assignment UI**: The waiting room `#renderPlayerList` shows only name + HOST badge. There is no way for the host to assign players to teams.

The fix uses a `#isHost` boolean field (set when `sendLobbyCreate()` is called, reset in `#clearView()`) as the single source of host truth. This boolean is available immediately during the waiting room — it does not depend on `nm.localPlayerId`.

## Tasks

### Task 1: Add `#isHost` field and set/reset it correctly

**File:** `src/scenes/lobby-scene.ts`
**Action:** Edit

1. Add the `#isHost` field declaration after `#playerName`:
   ```typescript
   #isHost: boolean = false;
   ```

2. In `#showLobbyListView()`, update the CREATE LOBBY button callback to set the flag **before** emitting the socket event (so the flag is ready when the `lobby:created` response arrives and triggers `#showWaitingRoomView`):
   ```typescript
   const createBtn = this.#createButton(cx, 100, 'CREATE LOBBY', () => {
     this.#isHost = true;
     NetworkManager.getInstance().sendLobbyCreate(this.#playerName);
   });
   ```

3. **Do NOT reset `#isHost` in `#clearView()`** — `#clearView()` is called at the top of every view transition including `#showWaitingRoomView()`, so resetting there would clear the flag before the host check that renders the START button. Instead, reset `#isHost = false` only in `shutdown()` and in the `#onLobbyStarted` handler (i.e., when leaving the lobby scene entirely):

   In `shutdown()` add:
   ```typescript
   this.#isHost = false;
   ```

   In the `lobby:started` EventBus/socket handler that transitions to the game scene, add before `this.scene.start(...)`:
   ```typescript
   this.#isHost = false;
   ```

   Leave `#clearView()` unchanged — it must not touch `#isHost`.

### Task 3: Add team badges and host toggle buttons to `#renderPlayerList`

**File:** `src/scenes/lobby-scene.ts`
**Action:** Edit

Replace the entire `#renderPlayerList` method with an updated version that:
- Increases row height from 28 px to 36 px (to accommodate team controls)
- Adds a read-only team badge for all clients (text label: "A", "B", or "—" for unassigned, colored accordingly)
- Adds "A" and "B" clickable toggle buttons only for the host — wired to `NetworkManager.sendLobbyAssignTeam(player.id, team)`
- Highlights the active team button (filled color) vs inactive (dim color)

```typescript
#renderPlayerList(players: PlayerInfo[]): void {
  this.#playerListObjects.forEach((o) => o.destroy());
  this.#playerListObjects = [];

  const cx = this.cameras.main.centerX;
  const baseY = 120;
  const TINTS = [0xffffff, 0x00aaff, 0xff4444, 0x44ff44, 0xff44ff];

  players.forEach((player, i) => {
    const rowY = baseY + i * 36;
    const tint = TINTS[i % TINTS.length];
    const dot = this.add.rectangle(cx - 150, rowY + 8, 12, 12, tint);
    const name = this.add.text(cx - 130, rowY, player.name, FONT_SMALL_WHITE);
    const role = player.id === this.#currentLobby?.hostPlayerId
      ? this.add.text(cx + 30, rowY, '(HOST)', FONT_SMALL)
      : null;

    this.#playerListObjects.push(dot, name);
    if (role) this.#playerListObjects.push(role);

    if (this.#isHost) {
      // Host sees clickable Team A / Team B toggle buttons per row
      const isTeamA = player.team === 0;
      const isTeamB = player.team === 1;
      const nm = NetworkManager.getInstance();

      const btnA = this.#createButton(cx + 90, rowY + 8, 'A', () => {
        nm.sendLobbyAssignTeam(player.id, 0);
      });
      const btnB = this.#createButton(cx + 130, rowY + 8, 'B', () => {
        nm.sendLobbyAssignTeam(player.id, 1);
      });

      // Dim non-active team button so the current assignment is visually clear
      // #createButton returns a Container; tint its first child (the background rectangle)
      const bgA = (btnA as Phaser.GameObjects.Container).getAt(0) as Phaser.GameObjects.Rectangle;
      const bgB = (btnB as Phaser.GameObjects.Container).getAt(0) as Phaser.GameObjects.Rectangle;
      if (!isTeamA) bgA.setFillStyle(BTN_DISABLED);
      if (!isTeamB) bgB.setFillStyle(BTN_DISABLED);

      this.#playerListObjects.push(btnA, btnB);
    } else {
      // Non-host sees a read-only team badge
      const teamLabel = player.team === 0 ? 'Team A' : player.team === 1 ? 'Team B' : '—';
      const teamColor = player.team === 0 ? '#4488ff' : player.team === 1 ? '#ff4444' : '#888888';
      const badge = this.add.text(cx + 80, rowY, teamLabel, { ...FONT_SMALL, color: teamColor });
      this.#playerListObjects.push(badge);
    }
  });
}
```

Note: `BTN_DISABLED` (value `0x223366`) is already defined at the top of the file.
