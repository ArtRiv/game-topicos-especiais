<objective>
Optimize the WebRTC message path to eliminate the primary bandwidth and CPU bottleneck causing progressive latency with 3+ clients.

Purpose: The current 60 Hz full-state broadcast to every peer is the root cause of degradation. Reducing to 20 Hz with dirty-checking cuts outbound messages by 66-90%+. Optimizing the receive path (Map lookup instead of .find()) reduces per-message CPU cost.

Output: NetworkManager with 20 Hz tick, dirty-checking, O(1) player lookup, and togglable perf metrics.
</objective>

**config.ts changes:**
- Change `NETWORK_TICK_RATE_HZ` from `60` to `20` (per D-07, restoring original design)
- Add `export const NETWORK_DEBUG = false;` — toggle for network perf logging

**network-manager.ts changes — dirty-checking:**
- Add a private field `#lastSentSnapshot: PlayerUpdatePayload | null = null;`
- In `sendPlayerUpdate(payload)`, before calling `#broadcastUnreliable`, compare payload against `#lastSentSnapshot`:
  ```typescript
  sendPlayerUpdate(payload: PlayerUpdatePayload): void {
    if (this.#lastSentSnapshot &&
        this.#lastSentSnapshot.x === payload.x &&
        this.#lastSentSnapshot.y === payload.y &&
        this.#lastSentSnapshot.direction === payload.direction &&
        this.#lastSentSnapshot.state === payload.state &&
        this.#lastSentSnapshot.element === payload.element) {
      return; // nothing changed — skip send
    }
    this.#lastSentSnapshot = { ...payload };
    this.#broadcastUnreliable({ type: 'pos', ...payload });
  }
  ```
- Reset `#lastSentSnapshot = null` in `disconnect()` and `stopGameTick()`

**network-manager.ts changes — O(1) player lookup:**
- Add a private field `#socketToPlayerId = new Map<string, string>();`
- In the `lobby:started` handler (where `this.#matchPlayers = matchConfig.players`), build the Map:
  ```typescript
  this.#socketToPlayerId.clear();
  for (const p of matchConfig.players) {
    this.#socketToPlayerId.set(p.socketId, p.id);
  }
  ```
- In `#setupDataChannel`'s `onmessage` handler, replace the `.find()` call:
  ```typescript
  // BEFORE: const player = this.#matchPlayers.find((p) => p.socketId === fromSocketId);
  // AFTER:
  const playerId = this.#socketToPlayerId.get(fromSocketId) ?? fromSocketId;
  ```
  Then use `playerId` directly in the switch cases instead of `player?.id ?? fromSocketId`.
- Also clear `#socketToPlayerId` in `#closeAllPeerConnections()`.

**network-manager.ts changes — reduce object allocation on hot path:**
- In `#broadcastUnreliable`, the caller passes `{ type: 'pos', ...payload }` which allocates a new object every tick. Instead, pre-build the message string once and send it to all channels:
  The current pattern is fine since `JSON.stringify` is called once in `#broadcastUnreliable` and sent to all channels. No change needed here — the dirty-check above already eliminates most calls.
