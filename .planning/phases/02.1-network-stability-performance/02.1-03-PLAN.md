<objective>
Replace JSON serialization with MessagePack binary encoding on all WebRTC data channel messages (both client and server).

Purpose: JSON.stringify/parse is CPU-intensive and produces 40-60% larger payloads than binary alternatives. With the server relay forwarding messages between ~20 clients, reducing serialization overhead on both ends directly improves throughput and latency.

Output: All data channel traffic uses MessagePack binary encoding. JSON is no longer used for any game data channel message.
</objective>

@msgpack/msgpack API:
```typescript
import { encode, decode } from '@msgpack/msgpack';
// encode returns Uint8Array
const packed: Uint8Array = encode({ type: 'pos', x: 100, y: 200 });
// decode takes Uint8Array or ArrayBuffer
const obj = decode(packed) as DcMessage;
```

WebRTC DataChannel binary:
```typescript
// Set binaryType on DataChannel
channel.binaryType = 'arraybuffer';
// Send binary
channel.send(packed); // Uint8Array
// Receive binary
channel.onmessage = (e: MessageEvent<ArrayBuffer>) => {
  const msg = decode(new Uint8Array(e.data)) as DcMessage;
};
```
