<objective>
Replace the full WebRTC P2P mesh with a star topology where each client connects to the server via a single WebRTC connection, and the server relays messages between clients.

Purpose: The mesh scales O(N²) — at 20 players each client maintains 19 connections. Moving to a star topology reduces client connections to O(1) and eliminates the upload bandwidth bottleneck (client sends once, server fans out to N-1).

Output: Working SFU relay — clients send to server, server forwards to room members. All game events (position, spells, transitions) flow through the server relay.
</objective>

    2. Create `game-server/src/webrtc-relay.ts` with class `WebRTCRelay`:

       **Constructor:** Takes a reference to the GameRoom map (Map<string, GameRoom>) so it can look up which room a client belongs to.

       **Key state:**
       - `#peers: Map<string, { pc: PeerConnection; unreliable: DataChannel | null; reliable: DataChannel | null }>` — one entry per connected client, keyed by socketId
       - Reference to `gameRooms: Map<string, GameRoom>` and a `socketToLobbyId: Map<string, string>` for routing

       **Public methods:**
       - `createPeerConnection(socketId: string, lobbyId: string): PeerConnection` — Creates a new `node-datachannel.PeerConnection` for a client. Configures ICE with STUN server `stun:stun.l.google.com:19302`. Sets up `onLocalDescription` callback to emit the SDP back via a provided callback. Sets up `onLocalCandidate` callback similarly. Returns the PeerConnection.
       - `handleRemoteDescription(socketId: string, sdp: string, type: string): void` — Sets the remote description on the peer's PeerConnection.
       - `handleRemoteCandidate(socketId: string, candidate: string, mid: string): void` — Adds an ICE candidate.
       - `removePeer(socketId: string): void` — Closes PeerConnection and cleans up maps.

       **Data channel setup:**
       When PeerConnection fires `onDataChannel`, check channel label:
       - Label `'pos'` → store as unreliable channel for that peer
       - Label `'events'` → store as reliable channel for that peer

       **Message forwarding (core relay logic):**
       When a data channel receives a message:
       - Parse the message (JSON for now — Plan 03 switches to MessagePack)
       - Look up which lobbyId this socketId belongs to (via `socketToLobbyId`)
       - Look up the GameRoom for that lobbyId
       - Get all other socketIds in the room via `room.getOtherSocketIds(socketId)`
       - For each other socketId, find their peer entry and send the message on the matching channel type (pos messages → unreliable channel, spell/transition → reliable channel)
       - Prepend `playerId` to the message before forwarding (look up via GameRoom)

       **Metrics:**
       - Track `#msgForwarded` counter; log per second when debug mode enabled

    3. Export `WebRTCRelay` as the default export.

    const pc = new PeerConnection('client-{socketId}', { iceServers: ['stun:stun.l.google.com:19302'] });
    pc.onLocalDescription((sdp, type) => { /* send back via socket.io */ });
    pc.onLocalCandidate((candidate, mid) => { /* send back via socket.io */ });
    pc.onDataChannel((dc) => { dc.onMessage((msg) => { /* handle */ }); });

       **Remove** the existing `webrtc:offer`, `webrtc:answer`, `webrtc:ice` handlers that forward between peers.

       **Add** new signaling flow where the server IS the peer:
       ```typescript
       socket.on('webrtc:connect', ({ lobbyId }: { lobbyId: string }) => {
         // Client requests a WebRTC connection to the server
         const pc = relay.createPeerConnection(socket.id, lobbyId, {
           onLocalDescription: (sdp, type) => {
             socket.emit('webrtc:server-description', { sdp, type });
           },
           onLocalCandidate: (candidate, mid) => {
             socket.emit('webrtc:server-candidate', { candidate, mid });
           },
         });
       });

       socket.on('webrtc:client-description', ({ sdp, type }: { sdp: string; type: string }) => {
         relay.handleRemoteDescription(socket.id, sdp, type);
       });

       socket.on('webrtc:client-candidate', ({ candidate, mid }: { candidate: string; mid: string }) => {
         relay.handleRemoteCandidate(socket.id, candidate, mid);
       });
       ```
