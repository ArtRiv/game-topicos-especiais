# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Auth + Server Scaffold.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope

This phase has **three UI surfaces**:

| Surface | Description | Complexity |
|---------|-------------|------------|
| `login.html` | Standalone HTML/CSS OAuth entry page | **Primary deliverable** |
| `index.html` session check | JS-level redirect before Phaser boots — no new visual UI | Minimal / no visible state |
| Phaser main menu logout | Button label copy only — spacing/colors follow existing game conventions | Copy only |

**Not in scope:** No new Phaser scenes, no React/shadcn components, no CSS framework.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | Plain HTML/CSS — not React/Next.js/Vite-with-framework |
| Preset | not applicable | No component library |
| Component library | none | Vanilla HTML elements only |
| Icon library | none | No icon library — Google logo handled via official OAuth button guidelines |
| Font (game brand) | Press Start 2P | Exists at `public/assets/fonts/Press_Start_2P/` |
| Font (login page body) | system-ui, -apple-system, sans-serif | System font stack — maximizes readability, zero load cost |

> **shadcn gate:** Not applicable — this is not a React/Next.js/Vite-with-React project. No initialization needed.

---

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, fine inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, input padding |
| lg | 24px | Card internal padding |
| xl | 32px | Card gap from page center |
| 2xl | 48px | Vertical section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Google OAuth button minimum touch target is 44px height (accessibility requirement).

---

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Display | 28px | 400 | 1.2 | Press Start 2P | Game name "MAGES" on login page — brand anchor |
| Heading | 16px | 400 | 1.3 | Press Start 2P | Login card subtitle (e.g., "Sign in to play") |
| Body | 14px | 400 | 1.5 | system-ui | Informational text, policy notes |
| Label | 16px | 600 | 1.0 | system-ui | Button text ("Sign in with Google") |

> **Rationale:** Press Start 2P is used sparingly (display + heading only) for game identity. System font handles body/button for clean legibility. Press Start 2P at small sizes becomes unreadable — restrict to 16px minimum.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f1a` | Page background (dark navy-black — fits fantasy theme) |
| Secondary (30%) | `#1e1e2e` | Login card surface |
| Card border | `#2d2d44` | Subtle card outline |
| Accent (10%) | `#7c3aed` | Card focus ring, hover border on OAuth button |
| Text primary | `#e2e8f0` | All foreground text |
| Text secondary | `#94a3b8` | Subtitle, helper text |
| Destructive | `#ef4444` | Error message text (OAuth failure) |

Accent (`#7c3aed` — violet) reserved for:
- Login card focus ring on keyboard navigation
- OAuth button hover/focus border outline

**Google OAuth button:** Follow Google's brand guidelines — white background, Google multicolor logo, dark text `#3c4043`, 1px solid `#dadce0` border. Do NOT override with accent color (violates Google brand policy).

## States — login.html

| State | Visible Change |
|-------|---------------|
| Default | Card + button visible, no error message |
| Button loading | Button text changes to "Signing in…", button disabled, cursor: default |
| OAuth error | Error slot appears below button with error copy |
| Redirecting | No UI change — browser follows redirect immediately |

---

## States — index.html session check

| State | Visible Change |
|-------|---------------|
| Valid session | No visible state — Phaser boots normally |
| 401 / no session | JS redirect to `login.html` before Phaser canvas renders |

> **Decision (locked D-11):** No loading spinner or splash screen is needed for the session check. The existing `#d7d7d7` background from `index.html` is visible for the fraction of a second before redirect. No new visual element required.
